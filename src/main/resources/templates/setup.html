Welcome use StorageHome

Before you use StorageHome, we need you help to set up StorageHome. Don't worry, it's easy to finish!
There's a checklist you need to check your server with this list before your setup StorageHome:
  1. Make sure the database is work properly
  2. Make sure you have permission to allow StorageHome change your database, no only view it
  3. Check the options in application.properties all are valid and no temporary
  4. Make sure the server will not sudden power outages during installation
  5. Make sure the disk have enough space
  6. If you use docker to deploy, check the image version is latest or long term support version, that way you're less likely to have problems
  7. If you just run the executable file to deploy, please check your operating system is in the support list

Database setup

Before set administer, we need you help StorageHome setup database we need to use
Don't worry, it is really easy to set, before all, please open a connection to StorageHome use database with a shell
And make sure StorageHome use database charset is utf8mb4, then we can show and add some beautiful char and storage more data
The most important point is the script assumption your database name is storagehome, is your database name isn't it, please modify the script!
Let's setup the database! 

-- If you didn't set the database charset as `utf8mb4`, drop current database and recreate a new database use this command and reconfigure StorageHome config file
-- create schema storagehome collate utf8mb4_0900_ai_ci;
-- create `configs` table
create table if not exists storagehome.configs
(
    id    int auto_increment,
    name  text not null,
    value text not null,
    constraint configs_id_uindex
        unique (id)
);

alter table storagehome.configs
    add primary key (id);

-- create `users` table
create table if not exists storagehome.users
(
    uuid        varchar(32) not null,
    role        text        not null,
    username    text        not null,
    email       text        not null,
    create_time datetime    not null,
    password    text        not null,
    constraint users_uuid_uindex
        unique (uuid)
);

alter table storagehome.users
    add primary key (uuid);
